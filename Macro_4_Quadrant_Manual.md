# 🧭 橋水四大象限 — 宏觀週期判斷模組 (4-Quadrant MacroEngine)

本文件詳細說明 Bridgewater (橋水基金) 經典的 **All Weather 經濟四象限框架**。這個模組是達里歐投資哲學的地基，重點關注**「經濟成長 (Growth)」**與**「通貨膨脹 (Inflation)」**兩大因子的預期變化。

在我們 Dalio Cycle Sentinel 系統中，這套四象限基礎邏輯會與《大債務危機》的 6 階段結合，作為最基底的資產動態權重傾斜 (Tilt) 依據。

---

## 🗂️ 經濟四象限與判斷條件

根據經濟的四大變數組合，市場無時無刻不處於以下四個象限之一。系統藉由 FRED API 實時抓取的 CPI (消費者物價指數) 與 GDP (國內生產毛額) 來決定目前落在哪個象限。

### 1️⃣ 高增長 / 低通脹 (Goldilocks 金髮女孩經濟)
這是對資本市場最友善的環境，企業獲利快速成長，但央行又不需要為了壓制通膨而激烈升息。
* **判斷條件** (實務基準閾值)：
  - **GDP YoY > 2.5%** (高於潛在增長趨勢)
  - **CPI < 3.0%** (通膨受控)
* **💰 配置偏向 (Risk-On)**：
  - **提升 (↑)**：股票 (Stocks)、公司債 (Credit)、一般公債 (Bonds)。
  - **降低 (↓)**：黃金 (Gold)、大宗商品。
* *此階段應盡可能追求資本利得。*

### 2️⃣ 高增長 / 高通脹 (Overheating 經濟過熱)
經濟運轉得太快，需求大於供給，雖然企業帳面營收很好，但原物料與工資的上漲正逐漸反噬利潤，且央行會開始升息。
* **判斷條件**：
  - **GDP YoY > 2.5%**
  - **CPI > 4.5%**
* **💰 配置偏向 (Inflation Hedge)**：
  - **提升 (↑)**：大宗商品 (Commodities)、黃金 (Gold)、抗通膨債券 (TIPS)。
  - **降低 (↓)**：長天期一般公債 (長債因升息遭受重創)。
* *此階段原物料與抗通膨資產開始超越股市。*

### 3️⃣ 低增長 / 低通脹 (Deflationary Bust 通縮性衰退)
經濟陷入停滯或萎縮，市場需求疲弱導致物價下跌。為了救市，央行被迫啟動降息循環。
* **判斷條件**：
  - **GDP YoY < 1.5%** (甚至陷入負增長)
  - **CPI < 2.5%** (甚至陷入通縮)
* **💰 配置偏向 (Defensive / Policy Catching)**：
  - **提升 (↑)**：長中天期公債 (如 IEI, TLT)、現金。
  - **降低 (↓)**：股票 (Stocks，因企業盈餘衰退)、大宗商品。
* *此階段「長天期公債」是最佳避風港，除了保本還能賺取降息帶來的龐大資本利得。*

### 4️⃣ 低增長 / 高通脹 (Stagflation 停滯性通膨)
這是對傳統 60/40 股債組合最具毀滅性的象限（也是債務循環第四階段的一種特殊分支）。經濟停滯，企業不賺錢，但物價依然高漲，導致央行不僅不能降息救市，可能還要繼續升息。
* **判斷條件**：
  - **GDP YoY < 1.5%**
  - **CPI > 3.5%**
* **💰 配置偏向 (Hard-Assets Strategy)**：
  - **提升 (↑)**：黃金 (Gold)、短期票據 / 現金 (避開存續期風險)、抗通膨債券 (TIPS)。
  - **降低 (↓)**：股票 (Stocks)、長天期一般公債 (Long Bonds)。
* *此階段唯一能保護購買力的只有實體避險資產與極短天期的類現金。*

## 📌 實作邏輯與防禦機制提示

在 `fetch_data.py` 與前端引擎的實作中，這套宏觀判定具備以下技術韌性：

1. **核心指標來源**：
   - 使用 **FRED API** 作為最高指導數據源。
   - 抓取：`CPIAUCSL` (通膨年增率), `A191RL1Q225SBEA` 或 `GDP` (經濟成長), `T10Y2Y` (景氣衰退前瞻指標), `GFDEGDQ188S` (民間債務佔 GDP 比例)。
2. **⚠️ 數據滯後防禦 (Graceful Degradation)**：
   - 總體經濟數據通常有 1~3 個月的發布落後（例如 Q1 的 GDP 往往在 Q2 才會完全定案）。
   - 若程式偵測到 `local_data.json` 內的數值落後當前日期 **大於 30 天**，系統底層會自動改採**「移動平均平滑值 (Moving Average)」**或切換為**最近一期的預測趨勢值**進行推估。
   - 此時，系統儀表板的該項數據旁會亮起 `⚠️ (Data Lag)` 強制提醒使用者，確保配置調整不會因為短期數據的空窗期而失真或暴衝。
